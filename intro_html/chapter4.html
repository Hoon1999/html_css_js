form 태그에 대해서 공부합니다. 
    winform 의 컨트롤, 엑셀의 양식 컨트롤을 html에선 form 태그에서 구현합니다. 
    정확히는 form 태그에서 input 태그로 구현합니다. 

    form 태그의 기본적인 속성은 action 과 method 입니다. 
    action 은 동작을 수행할 파일의 주소, method 는 동작을 전달할 방식 즉 get / post 입니다. 
    
    action 에 할당되는(?) 파일들을 form-handler 라고 합니다. 
    get / post 개념은 이미 알고 있으니 패스
    
    input 태그의 type 속성으로 구현할 기능을 지정합니다.

    filedset 그냥 그거네.. 그... 그룹박스

    아무튼 뭐 button/submit, filedset 말고는 뭐 특이사항은 없다.

    이미 기초적인 지식은 다 알고 있다... 애초에 모르면 뭘 만들수가 없더만

    CSS 스타일 작성할 때,

    클래스로 스타일 지정하는 것의 각각 장단점이나 쓰임새를 좀 
    알고싶은데.. 그런 내용은 없네

    inline css
    style 태그로 감싸서 스타일 적용시키는 방법.
    스타일의 속성 값으로 배경색 등..을 지정한다. 

    internal CSS
    head 부분에서 style 태그를 사용해 body 전체에 적용하는 방법. 
    당연히 id 또는 클래스, 태그이름을 지정하여 특정 태그만 스타일이 적용되도록 한다. 

    external css
    인터널CSS 를 별도의 파일로 작성한 것.
    작성 방법은 동일하다.
    
    자바스크립트 적용에 대해서 공부한다.
    자바 스크립트도 css 처럼 내부, 외부로 나뉜다. 
    음 근데 여기서 가르쳐 주는건 css로 비유하면 인라인 방식과 외부방식만 있다. 

    별도의 스크립트 파일에서 내용을 가져오려면(?) script 태그의 src 속성을 통해 가져온다. 
    음......여기선 body 뒤 에다가 작성해야한다는 내용은 없는데, 부트스트랩은 왜 /body 뒤에 작성해야 되는거지.

    noscript 태그는 script 태그와 함께 쓰인다. 
    일부 웹브라우저에서는 스크립트 태그를 사용할 수 없다.
    스크립트 태그가 작동하지 않으면, noscript 태그가 작동된다.
