HTML 의 모든 요소는 display 속성을 가지고 있다고 합니다.
display 속성은 block 과 inline 으로 구분됩니다.

block 은 해당 라인의 가로너비를 모두 차지하는 속성입니다. 위에서 작성한 table을 보면, table 밖에 오른쪽 공간이 텅 비어 있지만, 건너뛰고 table 아래부터 글이 작성됩니다. 이것이 block 입니다.

inline 은 block의 반대 개념입니다.
block 속성값의 대표적인 태그중 하나는 div 태그입니다.
div 태그는 여러 HTML 요소들을 묶는 데 사용됩니다.
여러 태그에 한번에 스타일을 적용시키고 싶다면, div로 묶은 뒤 해당 div에 스타일을 지정하면 됩니다.

inline 속성값의 대표적인 태그는 span 태그입니다.
div 태그의 반대입니다. 택스트 일부분에 속성을 적용시키고 싶다면, span 태그를 사용하면 됩니다.

        iframe 태그는 inline frame 이라는 의미입니다. 
        말 그대로 내부에 frame 을 생성합니다. 
        iframe 태그의 src 속성에 보여줄 컨텐츠의 주소를 지정합니다. 

        iframe 외에 frameset 태그를 이용하여 여러 frame을 띄울 수 있습니다. 
        하지만 HTML5 에서 권고하지 않는다고 합니다. 
        iframe 이나 CSS를 이용하여 표현해야합니다.

레이아웃을 작성할 때 사용하는 태그는 div와 시맨틱태그가 있다.
table 태그로 레이아웃을 작성하는것이 가능은 하지만...의미상 적합하지 않으므로 table 태그는 표를 작성할때만 사용하자.
table 태그대신 div 태그를 사용해 레이아웃을 주로 작성하였다고 한다.
div 태그의 단점은 div 태그는 id 속성으로 구분을 하는데, id 속성은 디자이너가 정하기 나름이라 웹 사이트마다 다른 id를 사용한다고 한다.
물론 암묵적인 약속이 있어 header 에 해당하는 부분은 header 로 작성하긴 한다.
하지만 그러한 가능성을 막아두는 역할도 하고, 애초에 모두가 header 라는 통일된 명칭을 사용한다면 하나의 새로운 요소로 만들어 버리는게 나을수도 있다.
그래서 나온게 시맨틱태그라고 한다.
